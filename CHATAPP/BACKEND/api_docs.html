Group Chat API Documentation

This backend provides a clean interface for a universal decoupled group chat application. All endpoints support cross-origin queries (CORS enabled) and accept requests formatted as either raw JSON (application/json) or traditional form-data.

Base URL Configuration:
Local Server Root: https://ruth.alwaysdata.net/BACKEND

🔐 User Registration

Creates a brand new identity inside the database system. Passwords undergo modern secure BCRYPT hashing natively on execution.

POST /Register.php

Payload Example

{
  "username": "example_user",
  "email": "user@gmail.com",
  "password": "securePassword123"
}

Responses

Success (200 OK):

{
  "status": "success",
  "message": "Account created!"
}

Duplicate / Integrity Errors (400 Bad Request):

{
  "status": "error",
  "message": "This username or email is already taken."
}

🔑 User Login

Validates account profiles and spins up a local tracking engine session instance block.

POST /Login.php

Payload Example

{
  "username": "example_user",
  "password": "securePassword123"
}

Responses

{
  "status": "success",
  "message": "Logged in successfully",
  "user": {
    "id": 12,
    "username": "example_user"
  }
}

🚪 User Logout

Clears active server state structures cleanly instantly.

POST /Logout.php

Responses

{
  "status": "success",
  "message": "Logged out successfully"
}

✉️ Send Group Message

Broadcasts an individual text data payload string onto the global shared chat board wall.

POST /send_message.php

Payload Example

{
  "sender_id": 12,
  "message_text": "Hello world, testing the new live chat feed!"
}

Responses

{
  "status": "success",
  "message": "Message sent"
}

🔄 Get Group Messages (Live Feed)

Pulls down the historical data timeline records. User profiles attach securely to arrays automatically.

GET /get_messages.php
💡 Frontend Execution Rule: Run a fetch query asynchronously targeting this file inside a standard JavaScript setInterval() block running continuously every 5000ms (5 seconds) to build dynamic real-time user feed interfaces.

Response Example (200 OK)

{
  "status": "success",
  "messages": [
    {
      "id": 1,
      "sender_id": 4,
      "username": "admin_dev",
      "message_text": "Database is running fine locally.",

    },
    {
      "id": 2,

      "username": "example_user",
      "message_text": "Hello world, testing the new live chat feed!",
